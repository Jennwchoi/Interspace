## Project

**Interspace**

Interspace is a collaborative canvas drawing app where users draw freehand strokes and the app responds with AI-generated analysis, line art reconstruction, and conversational responses using Google Gemini. It runs as a Vite/React frontend backed by Supabase Edge Functions (Deno) that proxy requests to the Gemini API. The app is used by a small team of collaborators, primarily in an NYU class context.

**Core Value:** Users can draw freely and receive meaningful AI responses — the drawing pipeline must work reliably end-to-end without crashing or rate-limiting errors.

### Constraints

- **Tech stack**: Vite + React + TypeScript + Supabase Edge Functions (Deno) — no changes to this
- **Deployment**: Supabase Edge Functions for backend; static host for frontend — no infrastructure changes
- **Scope**: Stabilization sprint only — no new features, no test suite introduction
- **API**: Gemini free tier — rate limiting is a real constraint; throttling must be conservative

## Conventions

## Naming Patterns
- React components: PascalCase `.tsx` — `DrawingCanvas.tsx`, `AnimatedBackground.tsx`, `DraggableSticker.tsx`
- Utility modules: camelCase `.ts` — `lumaRules.ts`, `visualAnalysis.ts`, `drawingDescription.ts`, `creativeApi.ts`
- Edge function handlers: camelCase `.tsx` — `gemini.tsx`
- Import/asset barrels: camelCase — `Colors.tsx`, `svg-59l4z78q9k.ts`, `svg-c4ittoo348.ts`
- Supabase server entry: `index.ts`
- Named exported functions use camelCase: `analyzeDrawingWithGemini`, `generateChatWithGemini`, `generateLineDrawingWithGemini`, `fetchWithBackoff`, `parseDataUrl`
- React components exported as `default export function PascalCase`
- Small inline helpers (non-exported) use camelCase: `extractPaletteFromResponse`, `getLumaResponse`
- `useState` variables: camelCase noun — `selectedTool`, `brushSize`, `currentColor`, `emotionPalette`
- Boolean state: prefixed `is`/`has`/`show`/`can` — `isLoading`, `isInitialExperience`, `hasDrawingChanged`, `showShapeDropdown`, `canvasHistory`
- Refs: suffixed `Ref` — `canvasRef`, `messagesEndRef`, `shapeDropdownRef`, `inactivityTimerRef`
- Constants: SCREAMING_SNAKE_CASE for module-level constants — `GEMINI_API_BASE`, `GEMINI_MODEL`, `LUMA_SYSTEM_PROMPT`, `INACTIVITY_TIMEOUT`, `DRAWING_CATEGORIES`, `SYMBOLS`, `ECHO_VOCABULARY`
- Interfaces: PascalCase, descriptive suffix — `VisionAnalysisRequest`, `VisionAnalysisResponse`, `ChatMessage`, `ChatRequest`, `LineDrawingRequest`, `LumaInput`, `StrokeData`, `ShapeData`, `FaceData`, `StickerData`, `ColorData`, `CompositionData`, `VisualMetrics`
- Union type literals for tool/mode values: `'pencil' | 'brush' | 'eraser' | 'fill' | 'eyedropper' | 'shape' | 'sticker'`
## TypeScript Usage
- `useState` typed explicitly when non-trivial: `useState<'pencil' | 'brush' | ...>('pencil')`, `useState<any[]>([])`
- Refs typed with handle interfaces: `useRef<DrawingCanvasHandle>(null)`, `useRef<HTMLDivElement>(null)`, `useRef<NodeJS.Timeout | null>(null)`
## React Patterns
- Single large `App` component in `src/App.tsx` holds all top-level state and orchestration logic. No state management library (Redux, Zustand, etc.) is used.
- Sub-components are in `src/components/` and receive props; canvas is controlled via an imperative handle (`forwardRef` + `useImperativeHandle`) — `DrawingCanvasHandle` in `src/components/DrawingCanvas.tsx`.
- `useState`, `useEffect`, `useRef` from React core
- `useImperativeHandle` / `forwardRef` for canvas handle
- `useDrop` from `react-dnd`
- `motion` / `AnimatePresence` from `motion/react` for animations
- Direct `fetch()` calls inside `async` functions, not wrapped in a custom hook or query library (no react-query / SWR)
- Pattern: `try/catch` with `console.error` + fallback return value
## Code Style
- 2-space indentation throughout
- Single quotes for string literals in `.tsx`/`.ts`
- Trailing commas in multi-line objects/arrays
- Semicolons present
- Arrow functions preferred for callbacks and non-component functions
- JSDoc-style block comments (`/** ... */`) used on exported utility functions in edge functions (`gemini.tsx`)
- Inline `//` comments used freely for section headers and intent explanations
## Import Organization
## Error Handling
- Throws are caught at the call site, which returns a hardcoded fallback string
- `error instanceof Error ? error.message : "Internal server error"` narrowing used in `index.ts`
## Logging
- `console.warn` for retryable conditions (rate limit backoff): `console.warn(\`Gemini 429 rate limit — retrying in ${delay}ms...\`)`
- `console.error` for API failures before re-throwing
- `console.log` for raw API responses during development: `console.log("Raw Gemini response:", raw)`
- Frontend uses `console.error` for failed fetches before returning fallback strings
## Comments
## Module Design
- Edge function modules (`gemini.tsx`) use named exports only; no default export
- React components use `export default function`
- Utility modules (`lumaRules.ts`, `visualAnalysis.ts`) use named exports for interfaces and functions
- Auto-generated imports (`svg-*.ts`, `Colors.tsx`) use default or named exports as dictated by the Figma export tool

## Architecture

## Pattern Overview
- Monolithic React frontend (`src/App.tsx`, 2381 lines) managing all UI state
- Supabase Edge Functions (Deno runtime) serving as a secure API proxy between the frontend and third-party AI services
- No database reads/writes detected — Supabase is used purely for hosting and invoking Edge Functions
- AI pipeline replaced: OpenAI/DALL-E/Replicate stubs exist (`openai.tsx`, `replicate.tsx`) but the active routing in `index.ts` delegates to Gemini 2.0 Flash for chat and vision, and Gemini 2.5 Flash for image generation
## Layers
- Purpose: All user interaction, tool state, drawing analysis, AI chat, and rendering
- Location: `src/App.tsx`, `src/components/`
- Contains: React functional components, all `useState`/`useRef`/`useEffect` hooks, inline fetch calls to the Edge Function
- Depends on: Edge Function API, `src/utils/`, `src/components/`, `src/imports/`
- Used by: End user (browser)
- Purpose: Encapsulated, reusable UI pieces
- Location: `src/components/`
- Contains: `DrawingCanvas.tsx` (canvas + sticker drop target), `AnimatedBackground.tsx`, `DraggableSticker.tsx`, `EmotionMeter.tsx`, `ColorWheel.tsx`, `CreativeModeSelector.tsx`, `ModelViewer.tsx`; Radix UI wrappers in `src/components/ui/`; Figma-exported components in `src/components/figma/`
- Depends on: `src/utils/`, Radix UI, motion/react, react-dnd
- Used by: `src/App.tsx`
- Purpose: Drawing analysis, AI input construction, visual metrics
- Location: `src/utils/`
- Contains:
- Depends on: Nothing external; pure TypeScript
- Used by: `src/App.tsx`
- Purpose: Secure backend proxy; holds API keys away from the browser; routes requests to Gemini and Replicate
- Location: `src/supabase/functions/server/`
- Contains:
- Depends on: Gemini API, Replicate API, `Deno.env` for secrets
- Used by: `src/App.tsx` via `fetch()`
## Data Flow
- All state is local React state (`useState`) inside `App.tsx` — no Redux, Zustand, or Context API used
- Canvas state managed imperatively via `canvasRef` (`DrawingCanvasHandle`) and an explicit history stack (`canvasHistory`, `historyIndex`)
- Idle timer (30 seconds inactivity) resets all state to initial experience
## Key Abstractions
- Purpose: Structured data object capturing all measurable drawing properties passed from canvas analysis to Luma AI description generators
- Examples: `src/utils/lumaRules.ts`
- Pattern: Plain TypeScript interface with numeric and typed fields; constructed by `DrawingCanvas.tsx` and consumed by `App.tsx`
- Purpose: Imperative ref API exposing canvas operations to the parent (`App.tsx`)
- Examples: `src/components/DrawingCanvas.tsx`
- Pattern: `forwardRef` + `useImperativeHandle` exposing `restoreState`, `exportDataUrl`, etc.
- Purpose: Thin wrapper functions that translate app-level requests into Gemini REST API calls
- Examples: `src/supabase/functions/server/gemini.tsx`
- Pattern: Named async exports (`analyzeDrawingWithGemini`, `generateChatWithGemini`, `generateLineDrawingWithGemini`) with typed request/response interfaces; retry logic via `fetchWithBackoff` for 429s
- Purpose: URL path-based routing; no framework router — uses `url.pathname.split("/").pop()` to match route segments
- Examples: `src/supabase/functions/server/index.ts`
- Pattern: Sequential `if` blocks on `path`; CORS headers applied to all responses including OPTIONS preflight
## Entry Points
- Location: `index.html` → `src/main.tsx` (inferred from Vite convention; `src/App.tsx` is the root component)
- Triggers: User opens browser
- Responsibilities: Mount React app, render `<App />`
- Location: `src/supabase/functions/server/index.ts`
- Triggers: HTTP request from the browser's `callLumaAI()` or canvas analysis fetch
- Responsibilities: CORS, routing, error handling, delegating to Gemini/Replicate handlers
## Error Handling
- Edge function catches all thrown errors and returns `{ error: message, fallback: "I am here. What would you like to share?" }` with HTTP 500
- `callLumaAI()` in `App.tsx` checks `errorData.fallback` and surfaces it as a chat message rather than throwing
- `analyzeDrawingWithGemini()` catches JSON parse failures and returns hardcoded safe defaults (`asciiArt: "○ · ○\n·   ·\n○ · ○"`, `echoKeywords: ["trace", "stillness", "space"]`)
- Gemini rate limit (HTTP 429) retried with exponential backoff (1s, 2s, 4s, 8s; max 3 retries) via `fetchWithBackoff()`
- Local fallback in `App.tsx`: `getLumaResponse()` generates rule-based responses when AI is unavailable
